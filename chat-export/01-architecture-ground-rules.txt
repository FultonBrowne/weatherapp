       51 +                                                              
       52 +**useEffect is a code smell in this architecture.** Almost eve
          +ry use case has a better alternative:                         
       53 +                                                              
       54 +| Instead of useEffect for... | Use this instead |            
       55 +|---|---|                                                     
       56 +| Fetching data | `useQuery` from TanStack Query |            
       57 +| Responding to state changes | Derive state during render, or

       58 +| Subscribing to external stores | `useSyncExternalStore` |   
       59 +| Running code on mount | Server Components (run on server), o

       60 +| Transforming data for rendering | Compute it inline or `useM

          +component |

      102 +## State Design Philosophy                                    
      103 +                                                              
      104 +**Derive, don't sync.** If a value can be computed from other 
          +state or props, compute it — don't store it in state and try t
          +o keep it in sync.
